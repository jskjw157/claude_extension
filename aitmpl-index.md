## 사용법

### 특정 컴포넌트 설치
```bash
# 에이전트 설치
npx claude-code-templates@latest --agent security/security-auditor

# 커맨드 설치
npx claude-code-templates@latest --command git/feature

# MCP 설치
npx claude-code-templates@latest --mcp database

npx claude-code-templates@latest --hook pre-tool

# 스킬 설치
npx claude-code-templates@latest --skill development

npx claude-code-templates@latest --setting global
```

## 템플릿 상세 분석

### Agent 구조 분석

#### code-reviewer.md (development-tools)
```yaml
---
name: code-reviewer
description: Expert code review specialist for quality, security, maintainability.
              Use PROACTIVELY after writing or modifying code.
tools: Read, Write, Edit, Bash, Grep
model: sonnet
---
```

**워크플로우:**
1. `git diff`로 최근 변경 확인
2. 변경된 파일에 집중 분석
3. 8가지 평가 영역 체크

**평가 영역:**
- 코드 명확성
- 네이밍 컨벤션
- 중복 방지
- 에러 처리
- 보안 (시크릿 노출)
- 입력 검증
- 테스트 적절성
- 성능 최적화

**피드백 구조:**
- Critical (즉시 수정)
- Warnings (중요 개선)
- Suggestions (선택적 개선)

---

#### debugger.md (development-tools)
```yaml
---
name: debugger
description: Debugging specialist for errors, test failures, unexpected behavior.
              Use proactively when encountering any issues.
tools: Read, Write, Edit, Bash, Grep
---
```

**5단계 디버깅 프로세스:**
1. 에러 메시지 & 스택 트레이스 수집
2. 재현 방법 파악
3. 실패 위치 특정
4. 최소 수정 적용
5. 해결책 검증

**출력 구조:**
- 근본 원인 설명
- 진단 증거
- 코드 수정안
- 테스트 방법
- 재발 방지 권장사항

---

#### security-auditor.md (security)
```yaml
---
name: security-auditor
description: Reviews code for vulnerabilities, implements secure authentication,
              ensures OWASP compliance.
tools: Read, Write, Edit, Bash
model: opus
---
```

**보안 영역:**
- JWT, OAuth2, SAML 인증
- OWASP Top 10 취약점
- CORS/CSP 설정
- SQL 인젝션 방지
- 암호화 (at rest & in transit)

**원칙:**
- Defense in depth (다중 보안 계층)
- Principle of least privilege
- 모든 입력 검증
- Secure failure modes

### Hook 구조 분석

#### backup-before-edit.json (pre-tool)
```json
{
  "type": "PreToolUse",
  "matcher": "Edit",
  "hooks": [{
    "command": "if [[ -f \"$FILE\" ]]; then cp \"$FILE\" \"$FILE.backup.$(date +%s)\"; fi"
  }]
}
```
**기능:** Edit 실행 전 타임스탬프 백업 생성

---

#### security-scanner.json (security)
```json
{
  "type": "PostToolUse",
  "matcher": ["Edit", "Write"],
  "hooks": [{
    "command": "semgrep + bandit + gitleaks + regex patterns"
  }]
}
```
**기능:** 파일 수정 후 자동 보안 스캔
- Semgrep: 정적 분석
- Bandit: Python 보안
- Gitleaks: 하드코딩된 시크릿 탐지

#### test-driven-development/SKILL.md
```yaml
---
name: test-driven-development
description: Use when implementing any feature or bugfix, before writing implementation code
---
```

**핵심 원칙:**
> "NO PRODUCTION CODE WITHOUT A FAILING TEST FIRST"

**Red-Green-Refactor 사이클:**
1. **RED**: 실패하는 테스트 작성
2. **Verify RED**: 실패 확인 (필수)
3. **GREEN**: 테스트 통과하는 최소 코드
4. **Verify GREEN**: 통과 확인
5. **REFACTOR**: 코드 정리 (green 유지)

## 핵심 패턴 비교: Monet vs AITMPL

| 측면 | Monet Registry | AITMPL |
|------|----------------|--------|
| **Agent 구조** | 상세 시스템 프롬프트 | 간결한 역할 정의 |
| **Hook 형식** | - | JSON 기반 |
| **Skill 구조** | SKILL.md + references | SKILL.md + scripts |
| **MCP** | nanobanana, sharp | DB, 브라우저, 웹 |
| **강점** | 이미지→컴포넌트 특화 | 범용 개발 도구 |
